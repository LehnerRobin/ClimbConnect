# Contributing to ClimbConnect

## Branch-Strategie
- `main`: stabile, getestete Versionen
- `dev`: Integrationsbranch
- Feature-Branches: `feature/<kurzbeschreibung>`

## Commit Messages (konventionell)
- `feat: …` neue Funktion
- `fix: …` Bugfix
- `docs: …` Dokumentation
- `refactor: …` Umstrukturierung ohne neue Funktion
- `test: …` Tests
- `chore: …` Wartung

## Pull Requests
- Beziehe dich auf eine Issue-Nummer (z. B. `Closes #12`)
- Checkliste erfüllen (Build, Lint, Tests)
